# Governance

## Maintainer

Origin is currently maintained by **@thupa-pro**.

## Decision Process

- **Bug fixes and documentation**: Maintainer approves directly.
- **New features and protocol changes**: Must go through the RFC process.
  Submit a pull request modifying RFC-0001.md or creating a new RFC document.
- **Design disputes**: Resolved by maintainer after community discussion.

## Releases

Origin follows semantic versioning. Releases are cut from `main` when the
maintainer determines the code is stable. Pre-release versions (alpha, beta,
rc) may be published for testing.

## Becoming a Committer

Committers are contributors who have demonstrated:
1. Consistent, high-quality contributions to the codebase or documentation
2. Understanding of the protocol's design principles
3. Reliable communication and collaboration

New committers are nominated by existing committers and approved by the
maintainer.

## Code of Conduct

This project follows the Contributor Covenant Code of Conduct
(see CODE_OF_CONDUCT.md). All participants are expected to uphold it.
